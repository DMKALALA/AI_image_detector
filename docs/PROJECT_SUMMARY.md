## ğŸ” Detection Methods

### Method 1: Deep Learning (BLIP + Modern Ensemble)
**Technology**: BLIP captioning + CNN ensemble (ResNet, EfficientNet, DenseNet)

**What it detects:**
- Visual inconsistencies in generated images
- Patterns learned from training on AI vs real images
- Artifacts in textures and edges

**Confidence**: High on clear AI-generated images
**Speed**: Fast (1-2 seconds)

### Method 2: Statistical Pattern Analysis
**Technology**: Pixel-level statistical analysis

**What it detects:**
- Color distribution anomalies
- Edge sharpness patterns
- Noise characteristics
- Frequency domain signatures

**Confidence**: Good for detecting statistical fingerprints
**Speed**: Very fast (<1 second)

### Method 3: Advanced Spectral & Forensics
**Technology**: DCT analysis + JPEG artifacts + inconsistency detection

**What it detects:**
- Spectral anomalies in frequency domain
- JPEG compression artifacts
- Clone/copy detection
- Noise inconsistencies
- Pixel value distribution outliers

**Confidence**: Excellent for manipulated images
**Speed**: Medium (2-3 seconds)
**Note**: Recently fixed overconfidence issue (was showing 100% incorrectly)

### Method 4: HuggingFace Specialist Ensemble â­
**Technology**: 3 fine-tuned Vision Transformers

**Models:**
1. **Umm-maybe/AI-image-detector** (ViT)
   - Trained on diverse AI generators
   - Good general-purpose detector
   
2. **abhishek/ai-vs-human-image-detector**
   - Specialized in distinguishing AI from human art
   - Strong on artistic styles
   
3. **Google ViT Base (AI Classifier)**
   - Fine-tuned on GenImage dataset
   - Excellent on modern generators (Midjourney, DALL-E, Stable Diffusion)

**Confidence**: **Highest weight (35%)** - Most reliable method
**Speed**: Medium (3-5 seconds on first run, faster after)
**Fine-tuning**: Can be fine-tuned on custom datasets

### Method 5: Enterprise Models Ensemble ğŸ†•
**Technology**: Commercial-grade model architectures

**Models:**
1. **Hive-style CNN (ResNet-50)**
   - Robust general-purpose classifier
   - Similar to Hive AI's approach
   
2. **Reality Defender-style (Swin Transformer)**
   - Advanced transformer architecture
   - Good at subtle manipulations
   
3. **CLIP-based Semantic Detector**
   - Detects semantic inconsistencies
   - Analyzes image-text alignment

**Confidence**: High for complex images
**Speed**: Medium (3-4 seconds)

### Ensemble Voting System

Results are combined using **weighted voting**:

```python
Default Weights:
```

**Dynamic Features:**
- Weights auto-normalize if methods unavailable
- Adaptive weighting adjusts based on real-world accuracy

## ğŸš€ Advanced Features

### 1. Feedback-Based Learning ğŸ§ 

**How it works:**
1. User uploads image â†’ Gets prediction
2. User marks prediction as correct/incorrect
3. System computes image hash (SHA-256)
4. Stores feedback in database + JSON file
5. On re-upload:
   - Detects same image via hash
   - Adjusts confidence based on learned feedback
   - Shows learning indicators
   - Builds training dataset

**Benefits:**
- Never makes same mistake twice (on same image)
- Continuously improves from real-world usage
- Creates high-quality training data
- Transparent learning process

**Implementation:**
- `detector/feedback_learning.py`: Core learning logic
- `feedback_memory.json`: Persistent storage
- Image hash deduplication
- Confidence boost/penalty system

### 2. Adaptive Weighting System ğŸ“Š

**How it works:**
1. Tracks accuracy of each method
2. Every 100 detections with feedback:
   - Calculates real-world accuracy per method
   - Adjusts weights proportionally
   - Smooths changes (70% new, 30% old)
   - Saves to `method_weights_config.json`
3. Future predictions use updated weights

**Benefits:**
- Methods that perform well get more influence
- Poor-performing methods downweighted
- Automatic optimization over time
- No manual tuning needed

**Implementation:**
- `detector/adaptive_weighting.py`: Weight calculation
- `detector/models.py`: MethodPerformance tracking
- Smooth transitions prevent instability

### 3. Auto-Retraining Trigger ğŸ”„

**How it works:**
1. Monitors feedback count
2. When reaches threshold (default: 100 samples):
   - Triggers `retrain_from_feedback` command
   - Runs in background (doesn't block UI)
   - Processes correct + incorrect feedback
   - Creates balanced dataset
   - Generates training manifest
3. Updates marker file to prevent duplicates

**Configuration:**
- `min_feedback_samples`: 50 (minimum before first retrain)
- `retrain_interval`: 100 (retrain every N new feedbacks)

**Benefits:**
- Fully automatic model improvement
- No manual intervention needed
- Uses real-world data
- Handles class imbalance

**Implementation:**
- `detector/auto_retrain.py`: Trigger logic
- `detector/management/commands/retrain_from_feedback.py`: Dataset creation
- Background subprocess execution

### 4. Fine-Tuning Pipeline ğŸ¯

**Datasets Supported:**
- **GenImage** (chosen): 1.3M images, modern generators
- **WildFake**: Diverse, older generators

**Process:**
```bash
# 1. Prepare dataset (splits into train/val/test)
python manage.py prepare_genimage_dataset --dataset-path /path/to/genimage

# 2. Fine-tune models (all 3 HuggingFace models)
python manage.py finetune_hf_models --model vit --epochs 3
python manage.py finetune_hf_models --model human_detector --epochs 3
python manage.py finetune_hf_models --model classifier --epochs 3

# 3. Models automatically loaded from hf_finetuned_models/
```

**Features:**
- Automatic dataset splitting (70/15/15)
- Early stopping to prevent overfitting
- Model checkpointing (best validation accuracy)
- Progress logging
- GPU acceleration support

**Implementation:**
- `detector/management/commands/prepare_genimage_dataset.py`
- `detector/management/commands/finetune_hf_models.py`
- `hf_finetuned_models/`: Saved model weights

### 5. Dynamic Weight Normalization

**Problem Solved:**
If Method 4 (HuggingFace) unavailable (e.g., missing models), weights don't sum to 100%.

**Solution:**
- Detects which methods are active
- Redistributes weights proportionally
- Ensures total = 100%
- Logs normalization for transparency

**Example:**
```
Original: M1=22%, M2=20%, M3=3%, M4=35% (unavailable), M5=20%
Active: M1, M2, M3, M5 (total = 65%)
Normalized: M1=34%, M2=31%, M3=5%, M5=31%
```

### 6. Conditional Model Imports

**Problem Solved:**
PyTorch requires shared memory (SHM), which fails in sandboxed test environments.

**Solution:**
- Environment variable: `ENABLE_MODEL_IMPORTS`
- When `0`: Skips PyTorch/model imports
- When `1` or unset: Normal operation
- Service returns `None` if models disabled
- Views handle `None` gracefully (503 error)

**Use Cases:**
- Running tests in constrained environments
- Quick code checks without loading models
- CI/CD pipelines

**Implementation:**
- `detector/three_method_detection_service.py`: Conditional imports
- `detector/tests.py`: Auto-sets `ENABLE_MODEL_IMPORTS=0`

## ğŸ”’ Security Features

### 1. API Authentication
- **X-API-Key header**: Required for all API endpoints
- **Bearer token**: Alternative authentication
- **Configurable**: Set via `API_KEY` environment variable
- **Error handling**: 403 if invalid, 503 if not configured

### 2. File Validation
**Multi-layer validation:**
1. **Size check**: Max 10MB
2. **MIME type check**: Must be image/*
3. **Extension whitelist**: .jpg, .jpeg, .png, .webp, .gif
4. **PIL verification**: Opens with Pillow to verify integrity
5. **Dimension limits**: Max 10000x10000 pixels
6. **Format validation**: Ensures valid image format

**Implementation**: `detector/security_utils.py` â†’ `validate_image_file()`

### 3. Filename Sanitization
- Removes path traversal attempts (../)
- Strips directory separators
- Removes special characters
- Adds timestamp to prevent collisions
- Preserves safe characters and extension

**Implementation**: `detector/security_utils.py` â†’ `sanitize_filename()`

### 4. CSRF Protection
- Enabled by default for web forms
- `@csrf_exempt` removed from write endpoints
- API endpoints use API key auth (exempt from CSRF)
- Protects against cross-site attacks

### 5. Secure Configuration
- **SECRET_KEY**: Required via environment variable
- **DEBUG**: Defaults to False in production
- **ALLOWED_HOSTS**: Explicit whitelist
- **Sensitive data**: Never committed (in .gitignore)

### 6. Error Handling
- Generic error messages to users (no stack traces)
- Detailed logging for debugging
- Graceful degradation (e.g., if models unavailable)
- HTTP status codes: 400, 403, 503 as appropriate
